\doxysubparagraph*{深度优先遍历}

A backtracking algorithm for stepping through a tree（回溯法）\+:
\begin{DoxyItemize}
\item At any node, proceed to the first child that has not yet been visited
\item If we have visited all the children (of which a leaf node is a special case), backtrack to the parent and repeat this process
\end{DoxyItemize}

Each node is visited multiple times in such a scheme（节点会被多次访问）
\begin{DoxyItemize}
\item First time\+: before any children
\item Last time\+: after all children, before backtracking
\end{DoxyItemize}

\doxysubparagraph*{拓扑排序（\+Topological Sort）(\+A\+O\+V)}

\begin{quote}

\doxysubparagraph*{Motivation}

Dependency between tasks\+: one task is required to be done before the other task can be done Dependencies form a partial ordering
\begin{DoxyItemize}
\item A partial ordering on a finite number of objects can be represented as a directed acyclic graph (D\+AG) \doxysubparagraph*{Task}
\end{DoxyItemize}

Given a set of tasks with dependencies, is there an order in which we can complete the tasks? \doxysubparagraph*{Theorem\+:}

A graph is a D\+AG if and only if it has a topological sorting Proof strategy\+: Such a statement is of the form a ↔ b and this is equivalent to\+: a → b and b → a \doxysubparagraph*{lemmas}

First, we need a two lemmas\+:
\begin{DoxyItemize}
\item A D\+AG always has at least one vertex with in-\/degree zero
\begin{DoxyItemize}
\item That is, it has at least one source
\end{DoxyItemize}
\item Any sub-\/graph of a D\+AG is a D\+AG \doxysubparagraph*{Algorithm}
\end{DoxyItemize}

Idea\+:
\begin{DoxyItemize}
\item Given a D\+AG V, iterate\+:
\begin{DoxyItemize}
\item Find a vertex v in V with in-\/degree zero
\item Let v be the next vertex in the topological sort
\item Continue iterating with the vertex-\/induced sub-\/graph V \textbackslash{} \{v\}
\end{DoxyItemize}
\end{DoxyItemize}

implement\+:
\begin{DoxyItemize}
\item Use a queue (or other container) to temporarily store those vertices with in-\/degree zero
\item Each time the in-\/degree of a vertex is decremented tozero, push it onto the queue
\end{DoxyItemize}

迭代算法一般较为常见，递归算法更为直观。因为递归算法需要大量空间资源，所以经常需要将其改写成迭代算法，这里为体现算法的思想，我主要考虑以递归为出发点的思想来理解算法。\hypertarget{md___users_yanxinhao__synology_drive__d_s_a__r_e_a_d_m_e_autotoc_md82}{}\doxyparagraph{迭代算法实例}\label{md___users_yanxinhao__synology_drive__d_s_a__r_e_a_d_m_e_autotoc_md82}

$>$当递归调用是整个函数体中最后执行的语句且它的返回值不属于表达式的一部分时，这个递归调用就是尾递归。尾递归函数的特点是在回归过程中不用做任何操作，这个重要特性很重要，因为大多数现代的编译器会利用这种特点自动生成优化的代码。当编译器检测到一个函数调用是尾递归的时候，它就覆盖当前的活动记录而不是在栈中去创建一个新的。编译器可以做到这点，因为递归调用是当前活跃期内最后一条待执行的语句。于是当这个调用返回时栈帧中并没有其他事情可做，因此也就没有保存栈帧的必要了。通过覆盖当前的栈帧而不是在其之上重新添加一个，这样所使用的栈空间就大大缩减了，这使得实际的运行效率会变得更高。因此，只要有可能我们就需要将递归函数写成尾递归的形式。

$>$朴素的递归算法(减而治之或分而治之)之所以效率低，是因为他们反复求解相同的子问题。

动态规划的原理\+:
\begin{DoxyItemize}
\item 最优子结构性质 \+: 问题的最优解由相关子问题的最优解组合而成，而这些子问题可以独立求解。

如何发觉最优子结构?这里给出一个通用模式\+:
\begin{DoxyEnumerate}
\item 做出一个选择
\item 假定已经知道了这种选择(先不关心这种选择如何得到)
\item 考虑这种选择会产生哪些子问题, 以及如何最好刻画子问题空间
\item 证明构成原问题最优解的组成部分(即每个子问题)的解就是它本身的最优解(一般是反证法)
\end{DoxyEnumerate}
\item 重叠子问题 \+:
\item 重构最优解
\end{DoxyItemize}

动态规划有两种等价的实现方法\+:
\begin{DoxyItemize}
\item 带备忘的自顶向下法(top-\/down with memoization)
\item 自底向上法(bottom-\/up method)
\end{DoxyItemize}

\begin{DoxyItemize}
